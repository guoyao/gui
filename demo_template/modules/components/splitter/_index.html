Usage

Enable splitters via JavaScript:

$('.gui-splitter').guiSplitter();

Markup

Data attributes are integral to the splitter plugin. Check out the example code below for the various markup types.

Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-min-size="".

Name type default description
minSize integer 0 If minSize specified, the width of the first part can't be less than minSize when drag splitter. Use data-min-size for data attributes.
maxSize integer (Total Size) If maxSize specified, the width of the first part can't be greater than maxSize when drag splitter. Use data-max-size for data attributes.
splitPosition integer / percentage No the initialized position of splitter or the initialized width of first part. (should between minSize and maxSize if specified) Use data-split-position for data attributes.

Methods

.guiSplitter(options)

Activates your content as a splitter component. Accepts an optional options object. 

$('#mySplitter').guiSplitter({
  minSize: 50,
  maxSize: 200,
  splitPosition: "50%"
})

.guiSplitter('show')

Show first part of splitter component

.guiSplitter('hide')

Hide first part of splitter component

.guiSplitter('toggle')

Toggle first part of splitter component to shown or hidden.

Events

GUI's splitter class exposes a few events for hooking into splitter functionality.

Event Type Description
start.gui.splitter This event fires immediately when the show or hide or toggle instance method is called. (Namely the jquery animation of splitter start.)
complete.gui.splitter This event is fired when jquery animation of splitter complete.
